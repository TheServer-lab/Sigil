# Contributing to Sigil

Thank you for your interest in contributing to Sigil! Your work helps improve the project for everyone.

Please read this file before submitting a pull request.

## Ways to contribute
- Report bugs with clear reproduction steps
- Propose features via issues and design discussions
- Improve or expand documentation
- Submit code changes (bug fixes, improvements, tests)
- Create plugins, examples, and tutorial material

## Project philosophy
Sigil is a steward-driven project with a single long-term maintainer and a focused design vision. Contributions are welcome, but not all changes will be accepted.
